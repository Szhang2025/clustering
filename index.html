What Is Clustering? unsupervised

Unsupervised Learning

Clustering finds natural groupings in data without labels. Unlike classification, there is no "correct answer" — we discover structure that already exists. The goal: points in the same cluster should be similar to each other and different from points in other clusters.

Distance Metrics

Euclidean: d(a,b) = √Σ(aᵢ−bᵢ)² Manhattan: d(a,b) = Σ|aᵢ−bᵢ| Cosine: d(a,b) = 1 − (a·b)/(‖a‖‖b‖) Chebyshev: d(a,b) = max|aᵢ−bᵢ|

Choice of distance defines what "similar" means — Euclidean is most common, but Manhattan handles outliers better.

Three Paradigms

Partitional (K-Means): divide into K non-overlapping clusters. Needs K specified upfront. Fast, works well on spherical clusters.

Density-based (DBSCAN): finds clusters as dense regions separated by sparse regions. Handles arbitrary shapes, finds outliers.

Hierarchical (Agglomerative): builds a tree of nested clusters. No K needed upfront — cut the tree at any level.

Evaluation (No Labels)

Inertia (WCSS): Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖² (lower = better) Silhouette score: s(i) = (b(i)−a(i)) / max(a(i),b(i)) a = mean intra-cluster dist b = mean nearest-cluster dist s ∈ [−1, 1], higher = better
Choosing the Right Algorithm

K-Means — When to Use

Works best on: compact, spherical clusters of similar size and density. Sensitive to outliers. Must specify K. Very fast — O(nKt) per iteration. Good first choice for large datasets when cluster shapes are simple.

DBSCAN — When to Use

Works best on: irregularly shaped clusters, data with noise/outliers. Automatically detects K. Sensitive to ε and minPts hyperparameters. Struggles with varying densities. Ideal for geographic data, anomaly detection.

Hierarchical — When to Use

Works best on: small-to-medium datasets where you want to explore structure at multiple scales. Produces a dendrogram for interpretability. Slow — O(n²) or O(n² log n). Great for gene expression, document similarity.

The Curse of DimensionalityCurse of Dimensionality
In high dimensions, all pairwise distances converge to the same value — everything becomes "equally far apart". Clustering breaks down. Solutions: dimensionality reduction (PCA, t-SNE) before clustering, or use cosine similarity.Wikipedia ↗

Distance-based clustering degrades in high dimensions. Always consider normalizing features and reducing dimensions first. The silhouette score itself becomes unreliable beyond ~20 dimensions.

K-Means — Step-by-Step Animation

Algorithm

1. Initialise K centroids (random or k-means++) 2. Assign each point to nearest centroid 3. Update centroids ← mean of assigned points 4. Repeat until centroids stop moving Converges in O(nKt) · t = iterations

K-Means++ Initialisation

Random init can get stuck in bad local minima. K-Means++ spreads initial centroids apart: first centroid is random; each subsequent centroid is chosen with probability proportional to d² (squared distance to nearest existing centroid). This typically reduces iterations by 2–5×.

P(x chosen as centroid) ∝ d(x, nearest_existing)²

DBSCAN — Density-Based Spatial Clustering

Algorithm

For each point p: Find all points within distance ε (ε-neighbourhood) If |N(p)| ≥ minPts → p is a core point Expand cluster by density-reachability Noise: not reachable from any core point Border: in ε-nbhd of core, not core itself

Core, Border, Noise

Core point: has ≥ minPts neighbours within ε. Forms the backbone of a cluster.

Border point: within ε of a core point but has fewer than minPts neighbours itself.

Noise point: not within ε of any core point. Labelled −1. This is DBSCAN's killer feature — it explicitly identifies outliers.

Hierarchical Clustering — Dendrogram

Agglomerative Algorithm

Start: each point is its own cluster Repeat: merge the two closest clusters Stop: when only one cluster remains Result: a dendrogram (tree of merges) Cut at height h → k clusters

Linkage Methods

Single: min dist between clusters. Makes long "chaining" clusters.
Complete: max dist. Compact, similar-sized clusters.
Average: mean of all pairwise distances. Balance between single and complete.
Ward: minimises within-cluster variance increase. Usually best for spherical clusters.
